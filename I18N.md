# Admin Portal — i18n guide (Story 8.3)

SkyAgent Admin ships with Vietnamese as the default locale and English as
the fallback. Upstream gin-vue-admin's Chinese strings are being migrated
out of source and into locale bundles.

## Architecture

```
services/admin/
├── server/
│   ├── global/i18n/            # TOML-backed message bundle (Go)
│   ├── middleware/i18n.go      # Accept-Language -> gin.Context
│   ├── model/common/response/  # Ok*/Fail* helpers + *Code helpers
│   └── resource/i18n/
│       ├── vi-VN.toml
│       └── en-US.toml
└── web/
    └── src/i18n/
        ├── index.js            # vue-i18n v9 bootstrap
        └── locales/
            ├── vi-VN.json
            └── en-US.json
```

Key conventions:

- **Locale tags** use BCP-47 (`vi-VN`, `en-US`). The backend normaliser maps
  primary subtags (`vi`, `en`) onto the canonical form.
- **Default locale**: `vi-VN`. **Fallback**: `en-US`.
- **Key format**: lowercase, dot-separated, scoped by domain —
  `admin.<domain>.<section>.<label>`. Examples:
  `admin.common.create_failed`, `admin.user.welcome`, `admin.rbac.permission_denied`.
- **Template placeholders**: `{{.name}}` style on the backend,
  `{name}` style on the frontend (vue-i18n default).
- **Backend logs remain English** regardless of active locale.
  Only user-facing `msg` fields go through i18n.

## Adding a new key

1. Add it to **both** locale bundles (FE `src/i18n/locales/*.json` and/or
   BE `server/resource/i18n/*.toml`, depending on where it's used).
2. Reference it via `$t('admin.foo.bar')` in templates, `t('admin.foo.bar')`
   in Composition API script, or `response.FailWithCode(c, "admin.foo.bar")`
   in Go.
3. Run `make admin-i18n-lint` before committing.

## Lint gate

```bash
make admin-i18n-lint            # both scopes, fail on any CJK
make admin-i18n-lint-fe         # frontend only
make admin-i18n-lint-be         # backend only
node services/admin/scripts/extract-cjk.mjs --scope=all      # report without failing
node services/admin/scripts/extract-cjk.mjs --scope=be --json # machine-readable
```

The scanner is intentionally **read-only** — it never rewrites source.
Bulk conversion is iterative: pick a file from the report, translate by
hand (or dev-agent session), re-run.

## Tests

```bash
make admin-test-i18n                    # Go unit tests for i18n package + response helpers
cd services/admin/web && pnpm test      # (when FE test suite is wired up)
```
